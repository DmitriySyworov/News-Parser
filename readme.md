# News Parser & Analyzer

A high-performance service designed for automatic news parsing from a pre-defined pool of sources, featuring user authentication, custom link analysis, and comprehensive statistics tracking.

## Core Features
* 🤖 **Automated Parsing** — Extracts content from a managed link pool in Redis with a 24-hour TTL, followed by long-term archiving in PostgreSQL.
* 🔐 **Secure Authentication** — Email-based authentication with stateless JWT generation.
* 🔗 **Custom Analytics** — Authenticated users can submit personal links for on-demand parsing and analysis.
* 📊 **Advanced Statistics** — Tracks global source popularity metrics alongside personal user CRUD history (created, updated, deleted, and restored articles).
* 📖 **Interactive API Docs** — Built-in Swagger UI for rapid API exploration.

## Quick Start (Docker Compose)

Ensure you have **Docker** and **Docker Compose** installed.

### 1. Clone the Repository
```bash
git clone https://github.com/DmitriySyworov/News-Parser.git
cd News-Parser
```

### 2. Configure Environment Variables
Copy the example environment file and fill in your custom credentials:
```bash
cp .env.example .env
```

### 3. Spin Up the Stack
```bash
docker compose up -d --build
```

### What Happens Behind the Scenes:
* **Pre-baked Chromium** — The headless browser is installed once during the Docker build stage. It initializes instantly without downloading anything from the internet at runtime.
* **Automated Database Migrations** — The `migrator` service automatically applies schema changes to PostgreSQL as soon as the database container is healthy.

### API Documentation
Once the containers are running, access the interactive Swagger documentation at:
🌐 **`http://localhost:8080/swagger/index.html`** *(or your configured external port)*

## Local Development (Without Docker for Go)

To modify and execute the Go codebase locally on your host machine while keeping infrastructure in containers:

### 1. Start Infrastructure Containers Only
```bash
docker compose up postgres redis -d
```

### 2. Update Local `.env` Pathing
Point your local environment file to `localhost` instead of the internal Docker bridge aliases:
```env
DSN=host=localhost user=admin password=secret_pass dbname=news_db port=5432
REDIS_ADDRESS=127.0.0.1:6379
```

### 3. Run the Go Application
```bash
go run cmd/main.go
```
*Note: In local mode, the application gracefully bypasses the `ROD_BIN` environment path and hooks directly into your machine's native browser install.*

## Production Resilience & Fault Tolerance

* **Environment Agnosticism ("Blind Config")** — Fail-safe initialization. If the physical `.env` file is omitted (as inside a production container), the application seamlessly sources configuration payloads directly from system environment variables.
* **Self-Healing Browser (Crash Recovery)** — Built-in panic interception (`recover`). If an isolated tab or the root Chromium process crashes under heavy JavaScript execution, the adapter disposes of the defunct process and re-initializes a fresh browser instance in under 1 second without halting the parsing workflow.
* **Zombie Process Mitigation** — Docker-native `init: true` orchestration utilizes a lightweight `tini` process controller. This continuously reaps orphan subprocesses generated by automated page cycling, keeping RAM thresholds stable over months of uptime.
